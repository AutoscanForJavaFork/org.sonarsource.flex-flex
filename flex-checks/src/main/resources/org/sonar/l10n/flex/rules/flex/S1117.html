Shadowing fields with a local variable or with a function parameter is a bad practice reducing code readability: It makes it confusing to know whether the field or the variable is and should be accessed.

Noncompliant Code Example

class Foo {
  public var myField:int;

  public function Foo(myField:int) {                   // Compliant - method/constructor parameters are not checked
    this.myField = myField;
  }

  public static function build(myField:int):Foo {                   // Compliant - static functions are not checked
    ...
  }

  public function setMyField(int myField):void{           // Compliant - setter methods are not checked
    this.myField = myField;
  } 

  public function set myField(int myField):void{           // Compliant - implicit setter methods are not checked
    this.myField = myField;
  } 

  public function doSomething():String {
    var myField:int = 0;                          // Non-Compliant - should be renamed
    ...
  }

  public function doSomethingElse(myField:int):String {   // Non-Compliant - should be renamed                 
    ...
  }
}
