In ActionScript 3, constructor code is always interpreted rather than compiled by the JIT at runtime that's why the body of constructors should be as light as possible. As soon as this body contains some branch (if, for, switch, ...) an issue is logged.

Noncompliant Code Example

public class Foo
{
  public function Foo()
  {
    ... //lot of logic and control flow statements
  }
}

Compliant Solution

public class Foo
{
  public function Foo()
  {
    init()
  }

  private function init():void
  {
    ... //lot of logic and control flow statements
  }
}
