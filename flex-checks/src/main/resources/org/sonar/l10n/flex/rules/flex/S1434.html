Creating a new variable with the type "Object" means that this variable may be used to store any kind of object. This feature might be useful in some specific contexts, but most of the time this is hazardous, and if there is future refactoring, the compiler will be unable to do any kind of type checking.

Noncompliant Code Example

var obj:Object = new String();
var foo = new Object();

Compliant Solution

var obj:String = new String();
var foo:IPortfolio = new Portfolio();
